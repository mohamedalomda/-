المسائل الرياضية والقواعد

📘 ثالثاً: القوانين أو المعدلات الرياضية

في هذا الدرس ما فيش معادلات رياضية أو قوانين عددية — لأنه درس تقني/برمجي/متعلق بمعايير الويب، مش رياضيات.

لكن في جزء مهم جدًا هو:

⚖️ القاعدة: Box Model Behavior in Quirks vs Standard Mode
✅ في Standard Mode:
width = content فقط
padding و border يزيدوا على العرض الكلي
❌ في Quirks Mode:
width = content + padding + border
يعني لو حددت width: 200px; و padding: 20px; و border: 5px;
فالعرض الفعلي = 200px فقط (الـ padding و border يتأكلوا من الداخل)
📌 شرح بسيط:
  • ده فرق جوهري جدًا في الـ CSS Layout.
  • في Standard Mode: العناصر بتتصرف كما هو محدد في المواصفات الحديثة.
  • في Quirks Mode: المتصفح بيعمل "تقدير" قديم عشان يحافظ على التوافق مع صفحات التسعينات.
  • ده بيسبب كوارث في التصميم لو مش عارف إنت فين.
💡 مثال تطبيقي:
styles.css
.box {
width: 200px;
padding: 20px;
border: 5px solid black;
}
  • في Standard Mode: العرض الكلي = 200 + 40 + 10 = 250px
  • في Quirks Mode: العرض الكلي = 200px (الـ padding و border يتأكلوا من الداخل)

الجزء النظري

3
🔹 يعني إيه Document Type Declaration؟
  • الـ مش تاج HTML، ده مجرد تصريح للمتصفح يقوله: "اسمع، الصفحة دي مكتوبة بالـ HTML5 أو الإصدار الفلاني."
  • المتصفح على أساسه بيحدد إزاي يعرض الصفحة.
🔹 أوضاع العرض (Rendering Modes):
1. Standard Mode:
  • المتصفح بيتبع المعايير (W3C / WHATWG).
  • كل حاجة زي الكتاب.
  • هو الوضع الطبيعي لو كتبت .
2. Quirks Mode:
  • وضع "التوافق مع الماضي".
  • لو نسيت تكتب DOCTYPE، أو كتبته غلط.
  • المتصفح بيرجع يتصرف زي إصدارات قديمة (IE5 تقريبًا).
  • مشاكل CSS كتير (box model بيتلخبط، margins/paddings مش زي ما متوقع).
3. Almost Standards Mode:
  • شبه Standard، بس في حتة صغيرة: بيعالج الصور داخل الـ table cells بطريقة قديمة.
  • موجود عشان صفحات قديمة متبوظش.
🔹 ليه الموضوع مهم؟
  • من غير DOCTYPE، الصفحة ممكن تبان مظبوطة في Chrome وتتلخبط في Safari.
  • لازم تبدأ أي صفحة بواحد صحيح:
html>

عرض الأكواد كما تظهر في المتصفح

عرض توضيحي لكيفية ظهور الأكواد في المتصفح

عندما يتم تنفيذ الكود في المتصفح، يظهر كالتالي:

I'm a properly formatted HTML5 page

This page contains a correct DOCTYPE and is displayed in Standard Mode

I might break in another browser

This page without DOCTYPE may appear differently in different browsers

As you can see, the content is the same but the display may vary depending on the presence or absence of DOCTYPE and the rendering mode the browser is operating in.

🔧 مشاكل برمجية:
1. نسيان كتابة ``
  • يؤدي لدخول المتصفح في Quirks Mode تلقائيًا.
  • التصميم ممكن يختلف من متصفح لمتصفح.
  • صعوبة في التحكم في الـ Layout.
2. كتابة DOCTYPE بطريقة خاطئة
  • مثلاً: `` أو `` (حساس لحالة الأحرف في بعض المتصفحات القديمة).
  • بعض المتصفحات ممكن تتعامل معه كأنه غير موجود → Quirks Mode.
3. استخدام DOCTYPE قديم بدون فهم
  • مثلاً HTML4 Transitional → بيسمح بأشياء مش مقبولة في HTML5.
  • بيأثر على الـ CSS و JavaScript في بعض الحالات.
4. عدم فهم تأثير الـ DOCTYPE على الـ CSS Box Model
  • المبرمج يحسب المساحات غلط → التصميم يطلع مشوه.
🛠️ مشاكل تقنية:
1. اختلاف العرض بين المتصفحات
  • Chrome ممكن يتعامل مع الصفحة بدون DOCTYPE بشكل شبه مقبول.
  • Safari أو Firefox أو IE ممكن يكسروها تمامًا.
2. مشاكل في التوافق مع الأجهزة أو الشاشات المختلفة
  • لأن الـ Layout مش مبني على معيار ثابت.
3. تأثير غير مباشر على SEO
  • لو الصفحة متكسرة أو فيها مشاكل عرض → المستخدم يخرج بسرعة → معدل الارتداد يزيد → ترتيب الصفحة يقل.
4. مشاكل في اختبارات Cross-Browser Testing
  • لازم تتأكد إنك في Standard Mode عشان الاختبار يكون دقيق.
✅ نصائح للحل:
  1. ابدأ كل صفحة HTML بسطر واحد فقط:
    html>
  2. استخدم أدوات المطور (DevTools) في المتصفح:
    • افتح Console → اكتب: document.compatMode
    • لو طلع "CSS1Compat" → فأنت في Standard Mode ✅
    • لو طلع "BackCompat" → فأنت في Quirks Mode ❌
  3. لا تحاول تصلح مشاكل الـ Layout قبل ما تتأكد من الـ DOCTYPE.
  4. لو شغال على مشروع قديم واستخدم DOCTYPE HTML4 — افهم الفرق بين Transitional و Strict.

مبتدئ (3 أسئلة)
1. ما وظيفة DOCTYPE في HTML؟
✅ يحدد للمتصفح طريقة عرض الصفحة (Rendering Mode).
2. ما هو DOCTYPE الخاص بـ HTML5؟
✅
3. ماذا يحدث لو نسيت DOCTYPE؟
✅ المتصفح بيدخل في Quirks Mode.
متوسط (4 أسئلة)
4. ما الفرق بين Standard Mode و Quirks Mode في الـ Box Model؟
✅ في Standard: width = content فقط.
✅ في Quirks: width = content + padding + border.
5. ما هو سبب وجود Almost Standard Mode؟
✅ للتوافق مع صفحات قديمة كانت بتستخدم صور داخل الجداول.
6. هل DOCTYPE تاج (Tag)؟
✅ لا، ده مش تاج HTML، ده Declaration.
7. هل DOCTYPE بيأثر على SEO؟
✅ مش مباشر، بس عدم وجوده ممكن يعمل مشاكل عرض تؤثر سلبًا على الترتيب.
متقدم (باقي الأسئلة)
8. كيف يتعامل محرك العرض مع صفحة بدون DOCTYPE مختلفة عن صفحة فيها DOCTYPE خاطئ؟
✅ الاتنين ممكن يدخلوا Quirks Mode، بس بعض المتصفحات بتفرق وتدخل Almost Standards في حالات معينة.
9. ما علاقة DOCTYPE بالـ CSS Parsing؟
✅ DOCTYPE بيحدد إن الـ CSS يتفسر بمعايير حديثة، وإلا ممكن يتفسر بمعايير قديمة.
10. هل ممكن أكتب DOCTYPE لـ HTML4.01؟
✅ أيوه:
"-//W3C//DTD HTML 4.01 Transitional//EN"
"http://www.w3.org/TR/html4/loose.dtd">
11. إيه الفرق بين Transitional و Strict DOCTYPEs في HTML4؟
✅ Transitional بيسمح بعناصر قديمة غير قياسية، Strict بيرفضها ويجبرك على المعايير.
